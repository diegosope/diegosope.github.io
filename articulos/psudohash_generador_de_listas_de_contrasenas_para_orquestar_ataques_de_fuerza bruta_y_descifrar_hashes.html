Psudohash: Generador de listas de contraseñas para orquestar ataques de fuerza bruta y descifrar hashes 📙💪

 

Sobre la herramienta:

Psudohash  es un generador de listas de contraseñas para orquestar ataques de fuerza bruta y descifrar hashes. Limita ciertos patrones de creación de contraseñas comúnmente utilizados por los humanos, como sustituir las letras de una palabra por símbolos o números (leet), utilizar variaciones entre mayúsculas y minúsculas, añadir un relleno común antes o después de la frase de contraseña principal y más. Se basa en palabras clave y es altamente personalizable.

 

Pentesting de entornos corporativos:

Los administradores de sistemas y otros empleados suelen utilizar una versión mutada del nombre de la empresa para establecer las contraseñas (por ejemplo, Am@z0n_2022). Este es comúnmente el caso para dispositivos de red (puntos de acceso Wi-Fi, switches, routers, etc), aplicaciones o incluso cuentas de dominio. Con las opciones más básicas, psudohash puede generar una lista de palabras con todas las posibles mutaciones de una o múltiples palabras clave, basadas en patrones comunes de sustitución de caracteres (personalizables), variaciones de mayúsculas y minúsculas, cadenas comúnmente usadas como relleno y más. Personalización:

Sustitución de carácteres Leet

El script implementa el siguiente esquema de sustitución de caracteres. Puede añadir/modificar mapeos de sustitución de caracteres editando la lista de transformaciones en psudohash.py y siguiendo la estructura de datos presentada a continuación (por defecto):

 

 

transformaciones = [

                {'a' : ['@', '4']},

                {'b' : '8'},

                {'e' : '3'},

                {'g' : ['9', '6']},

                {'i' : ['1', '!']},

                {'o' : '0'},

                {'s' : ['$', '5']},

                {'t' : '7'}

]

 

 

Valores de relleno comunes

Al establecer contraseñas, creo que es bastante habitual añadir una secuencia de caracteres antes y/o después de la frase de contraseña principal para hacerla más "fuerte". Por ejemplo, uno puede establecer una contraseña "dragón" y añadir un valor como "!!!" o "!@#" al final, resultando en "dragón!!!", "dragón!@#", etc. Psudohash lee dichos valores de common_padding_values.txt y los utiliza para mutar las palabras clave proporcionadas añadiéndolos antes (-cpb) o después (-cpa) de cada variación de palabra clave generada. Puede modificarlo a su gusto.

 

Valores anuales

Al añadir un valor de año a una palabra clave mutada, psudohash lo hará utilizando varios separadores. Por defecto, utilizará los siguientes separadores que puede modificar editando la lista year_seperators:

 

year_seperators = ['', '_', '-', '@']

 

Por ejemplo, si la palabra clave dada es "amazon" y se ha utilizado la opción -y 2023, la salida incluirá "amazon2023", "amazon_2023", "amazon-2023", "amazon@2023", "amazon23", "amazon_23", "amazon-23", "amazon@23".

 

 

Instalación:

 

git clone https://github.com/t3l3machus/psudohash

cd ./psudohash

chmod +x psudohash.py

 

En primer lugar, usaremos el siguiente comando para clonar el repositorio de psudohash, nos ubicarnos en la carpeta de la herramienta


❯ git clone https://github.com/t3l3machus/psudohash


 

❯ cd ./psudohash


 

y le daremos permisos para poder ejecutarla


❯ chmod +x psudohash.py



 

 

Uso:

 

  ./psudohash.py [-h] -w WORDS [-an LEVEL] [-nl LIMIT] [-y YEARS] [-ap VALUES] [-cpb] [-cpa] [-cpo] [-o FILENAME] [-q]

 

 

Tips de uso:

Consejos de uso

Combinando las opciones --years y --append-numbering con un --numbering-limit ≥ dos últimos dígitos de cualquier año introducido, es muy probable que se produzcan palabras duplicadas debido a los patrones de mutación implementados por la herramienta.

Si añade valores de relleno personalizados y/o modifica los valores de relleno comunes predefinidos en el código fuente, en combinación con múltiples parámetros opcionales, existe una pequeña posibilidad de que se produzcan palabras duplicadas. psudohash incluye controles de filtrado de palabras, pero, por motivos de velocidad, son limitados.

 

 

Individuos

Cuando se trata de personas, creo que todos tenemos (más o menos) contraseñas establecidas usando una mutación de una o más palabras que significan algo para nosotros, por ejemplo, nuestro nombre o los nombres de esposa/hijo/mascota/bandas, pegando el año en que nacimos al final o tal vez un relleno súper seguro como "!@#". ¿Y adivina qué?

 

Podemos mutar más de una palabra con la opción –w entrecomillándolas y separándolas por coma, con relleno común antes y después -cpb (common padding before) -cpa (common padding after), con valores anuales -y, con un nivel 3 -an3 a nuestro diccionario al que podemos nombrar con la opción -o


❯ ./psudohash.py -w ‘maria,jose,max,luffy’ -cpb -cpa –y 1993,1995,1997 -an 3 -o wordlist.txt


 

 

 

Comprobamos el número de líneas del diccionario, y además que dichas palabras no están repetidas.
